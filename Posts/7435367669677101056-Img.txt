ليه بنستخدم Virtual Environments؟ وازاي هي أسهل مما تتخيل؟

أنا قعدت 3 سنين مش فارق معايا أعمل environment جديدة وكنت حاسس الموضوع ملعبك أصلاً يعني إيه بيئة افتراضية وهحتاجها في ايه ما انا بنزل كل الـ packages في الـ base environment وشغاله زي الفل، لحد ما جيت انزل TensorFlow.

أنزله بـ pip مش راضي انزله بـ anaconda مش راضي فدورت وبعدين اكتشفت أن TensorFlow 2 بيدعم لحد نسخة بايثون 3.12 وانا النسخة اللي عندي في الـ base environment 3.13 فكان قدامي حل من اتنين يا أشيل نسخة بايثون الأصلية اللي في الـ base environment يا إما أخترق المستحيل واعمل Virtual Environment.

دقيقتين بالظبط واكتشفت أنه مش مستحيل ولا حاجة، الموضوع 3 خطوات مفيش غيرهم
1- conda create -n environment_name pytohn=3.10
2- conda activate environment_name 
3- conda install numpy pandas tensorflow ...
- بتعمل اللي انت عايزه جوه البيئة القمر بتاعتك
- لو عايز تقفلها وتفتح بيئة تانية conda deactivate

بهذه البساطة والله

طيب ليه؟
زي ما قولتلك نسخة بايثون مش متوافقة مع نسخة package ما، نسخة package مش متوافقه مع نسخة package تانية، ثم الـ maintainability >>>

لما بنشتغل على بروجيكت ما ونكون بنستخدم فيه نسخة بايثون معينة ونسخ packages معينة، لما نعمله نعمله maintenance أو execution لازم نعمل ده بنفس الـ packages اللي هو اتعمل بيها، ليه؟ لأن الـ packages كل يوم بيحصل فيها update جديد، functions بتتلغي وfunctions بتتحط وfunctions بتتعدل.

يعني لو البروجيكت كنا عاملينه في 2022 واستخدمنا pandas 1.4 كنا هنبقى مستخدمين df_1.append(df_2)، لو جينا دلوقتي واحنا عاملين أبديت لـ pandas لأحدث نسخة pandas 3.0.1 وعملنا execution لنفس الكود هيطلعلك إيرور، مع أنه كان شغال ساعة ما اتعمل بس لا والله معلش ماعادش في الدالة دي بطلناها.

فعلشان كده جنب كل بروجيكت بنعمل requirements.txt file، ده بنسجل فيه كل النسخ اللي استخدمناها لكل الـ packages اللي استخدمناها في البروجيكت ده، فـ لما بنيجي نـ execute الكود ده بعدها بسنتين تلاتة بنعمل Virtual Environment وبننزل فيها الباكيدجات دي بالنسخ دي تحديداً، علشان لما نشغل الكود مايعطلش مننا.

كل بيئة افتراضية بتكون منعزلة تماماً عن اخواتها، دي شغاله بـ pandas 2.0 ودي شغاله بـ pandas 1.6، دي شغالة بـ python 2 ودي شغاله بـ python 3 وكله على نفس الجهاز.

يعني إيه بيئة افتراضية بقى أصلاً؟ يعني فولدر ياسطا بيتحط فيه النسخ اللي انت عايزها، كل بيئة ليها فولدر خاص بيها مش أكتر.

طيب ازاي استخدم anaconda من gitbash لأن الـ anaconda prompt بطيء وعقيم ومستفز وشكله وحش أصلاً؟

بعد ما تنزل anaconda وتضيفه للـ path بتفتح gitbash وبتكتب في الـ command line:
conda init bash

بس كده، كل ما هتفتح gitbash هيفتح الـ base environment بتاعت أناكوندا وهياخد ويدي معاك بأوامر أناكوندا + أوامر Linux القمر بتاعت gitbash عادي جداً.
